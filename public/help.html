VizCode — Help & Usage

1. How to use the tool window

The app presents three main areas: an icon library (left), a code-style prompt editor (middle), and a diagram preview (right). Use the icon library to drag icons onto the canvas, edit the textual prompt to describe clusters/nodes/edges, or import an Excel file to build the diagram.

2. What to click — step by step

  1. To add a new cluster: type a name in Add Cluster and click Add.
  2. To add icons: drag any icon from the left panel to the canvas on the right.
  3. To create an arrow between two nodes: hold Shift, click and drag from one node to another.
  4. To remove nodes or edges: use Ctrl+Shift+Click on the target (this avoids accidental deletes).
  5. To rename a node label: double-click the node label (or double-click the node to open the inline editor).
  6. Use the Arrow Type selector to choose the default arrow style before creating new arrows.

Copying icon names

You can right-click any icon in the left Icons panel to copy its name to the clipboard immediately. There is no confirmation dialog—copy happens instantly. A short non-blocking message (toast) appears briefly in the lower-right corner to confirm the copy (for example: API Gateway copied).

Use this when composing prompts or building Excel node lists: right-click an icon, then paste the copied name into the prompt editor or into your spreadsheet's Node column.

3. Importing and updating

You can import an Excel file (.xlsx) with two sheets:

  • Sheet 1 (Nodes) — columns: Node, Name (optional), Cluster, Parent cluster (optional).
  • Sheet 2 (Edges) — columns: Source, Destination, Type (arrow style), Label (optional).

After importing the file the prompt is generated automatically and the diagram will update.

4. Writing the prompt and syntax

The prompt is indentation-driven. Use Cluster: NAME to declare clusters and indent child clusters or nodes beneath them using two spaces per indent level. Example: 

Cluster: Auth
  Node: User
  Node: System
  Cluster: Internal
    Node: DB
User -> System [arrow=solid, label='auth request']

Rules:

  • Cluster: NAME declares a cluster. Indent child clusters or nodes with two spaces.
  • Node: LABEL [name=alias] declares a node; name is an optional identifier used in edges.
  • A -> B [arrow=TYPE, label='text'] creates an edge. Supported arrow types: solid, dashed, double, double-dotted.
  • Edge label is optional and should be quoted if it contains spaces or special characters.

When editing the prompt, click Update Diagram to apply changes. The editor will also auto-parse on new lines.

5. Tips & troubleshooting

  • If nodes or outlines overlap, try re-running Update Diagram — the layout algorithm will re-flow clusters.
  • Hold Ctrl/Cmd + Shift while clicking nodes or arrows to delete safely.
  • Use the zoom controls in the preview to focus on dense areas before exporting.
